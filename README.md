## Mixed-Model Alloy Wheel Manufacturing Line Simulation 
This repository contains a high-fidelity, discrete-event simulation (DES) built using Python and SimPy to model, analyze, and optimize mixed-mode alloy wheel manufacturing system. 
Mixed-Model Alloy Manufacturing/ How orders are generated
Wheels orders are grouped into families with bach size is generated by random generator Production line operate as mixed-model lines. This means different wheel designs and sizes (ranging from compact $14''$ passenger wheels to massive $24''$ custom wheels) 
A setup change (or changeover) is the non-productive downtime required to reconfigure a machine so it can transition from processing one wheel family to another.

#** Operational Optimization & Risk Analysis**
To improve factory operations, two optimization studies were conducted:

Heat Treatment Capacity Optimization
Analyzed how the physical size of the Heat Treatment furnace impacts wait times.

The Problem: If the oven capacity is too small, it creates a severe physical bottleneck. If the oven capacity is too large, wheels sit idle waiting for a long time trying to gather a big enough batch to start the bake.

The Solution: By plotting the furnace capacity against the maximum queue wait times, found the exact physical "sweet spot" where the furnace runs at high capacity while keeping inventory waiting times to a minimum.

Cumulative Lead Time & Risk Analysis
Real-world factories experience continuous minor delays (machine jams, tool breaks, and setup changes) that add up over time.

The Methodology: I have ran the simulation 20 times using different random schedules to see how local delays behave.
